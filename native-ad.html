üì™ Native Ad

This page provides comprehensive instructions for implementing native advertisements within your Android application using the AdSter SDK. Native ads match the look and feel of your app's content.

Step 1: Create Ad Request Configuration

Initialize the configuration with your placement name:

Java
Kotlin
Java 
AdRequestConfiguration.Builder configuration =
    AdRequestConfiguration.Companion.builder(context, "Your_placement_name");
Kotlin 
val configuration = AdRequestConfiguration.builder(context, "Your_placement_name")

Step 2: Load the Advertisement

Implement the ad loader with listener callbacks for success and failure scenarios:

Java 
AdSterAdLoader.Companion.builder()
    .withAdsListener(new MediationAdListener() {
        @Override
        public void onNativeAdLoaded(@NonNull MediationNativeAd ad) {
            // Show native ad here
        }

        @Override
        public void onFailure(@NonNull AdError adError) {
            // Handle failure
        }
    })
    .withNativeAdEventsListener(new NativeAdEventsListener() {
        @Override
        public void onAdClicked() { }

        @Override
        public void onAdImpression() { }

        @Override
        public void onAdRevenuePaid(double revenue,
            String adUnitId, String network) { }
    })
    .build()
    .loadAd(configuration.build());

Step 3: Define Layout XML

Create a layout file containing UI elements for displaying ad content:

XML 
<com.adster.mediation.nativead.MediationNativeAdView
    xmlns:android="http://schemas.android.com/apk/res/android"
    android:id="@+id/native_ad_view"
    android:layout_width="match_parent"
    android:layout_height="wrap_content">

    <LinearLayout
        android:layout_width="match_parent"
        android:layout_height="wrap_content"
        android:orientation="vertical">

        <!-- Ad Title -->
        <TextView
            android:id="@+id/ad_headline"
            android:layout_width="match_parent"
            android:layout_height="wrap_content" />

        <!-- Ad Body -->
        <TextView
            android:id="@+id/ad_body"
            android:layout_width="match_parent"
            android:layout_height="wrap_content" />

        <!-- Media View -->
        <FrameLayout
            android:id="@+id/ad_media"
            android:layout_width="match_parent"
            android:layout_height="200dp" />

        <!-- Call to Action Button -->
        <Button
            android:id="@+id/ad_call_to_action"
            android:layout_width="wrap_content"
            android:layout_height="wrap_content" />

    </LinearLayout>

</com.adster.mediation.nativead.MediationNativeAdView>

Step 4: Display the Native Ad

üí°
Critical Requirements
  • Call trackViews() before adding the view to container
  • Call setNativeAd() method before rendering
  • Use MediationNativeAdView as parent container
Java 
@Override
public void onNativeAdLoaded(@NonNull MediationNativeAd ad) {
    // Inflate your native ad layout
    MediationNativeAdView adView = (MediationNativeAdView) getLayoutInflater()
        .inflate(R.layout.native_ad_layout, null);

    // Set ad content to views
    TextView headline = adView.findViewById(R.id.ad_headline);
    headline.setText(ad.getHeadline());

    TextView body = adView.findViewById(R.id.ad_body);
    body.setText(ad.getBody());

    Button cta = adView.findViewById(R.id.ad_call_to_action);
    cta.setText(ad.getCallToAction());

    // Set native ad and track views
    adView.setNativeAd(ad);
    ad.trackViews(adView, mediaView, clickableViews);

    // Add to container
    container.removeAllViews();
    container.addView(adView);
}

Step 5: Clean Up Resources

Call MediationNativeAd.destroy() during activity or fragment destruction:

Java 
@Override
protected void onDestroy() {
    super.onDestroy();
    if (nativeAd != null) {
        nativeAd.destroy();
    }
}

Key Callbacks

Callback Description
onNativeAdLoaded() Triggered when ad loads successfully
onFailure() Handles error conditions
onAdClicked() Tracks user interactions
onAdImpression() Records ad impressions
onAdRevenuePaid() Reports revenue data

Tips
  • Ensure proper view attachment order to maintain tracking functionality
  • Use image loading libraries (e.g., Glide) for logo rendering
  • Set layout parameters matching parent container specifications
